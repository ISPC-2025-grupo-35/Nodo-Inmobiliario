Git es un sistema de control de versiones distribuido. Fue creado por Linus Torvalds en 2005 con el objetivo de gestionar el desarrollo del núcleo de Linux. Su función principal es permitir que múltiples personas trabajen de forma simultánea y segura sobre el mismo proyecto, manteniendo un historial completo de los cambios realizados en el código. Git permite deshacer errores, comparar versiones, y trabajar en paralelo mediante ramas (branches), facilitando un flujo de trabajo ordenado y colaborativo.

A diferencia de otros sistemas de control de versiones, Git no depende de una conexión constante con un servidor central: cada desarrollador tiene una copia completa del repositorio, incluyendo todo el historial. Esto no solo mejora la velocidad y eficiencia, sino que también reduce el riesgo de perder información. Las operaciones más comunes —como commits, merges, o cambios de rama— se realizan localmente, lo que hace a Git especialmente robusto y flexible.

GitHub, por su parte, es una plataforma web que permite alojar repositorios Git de forma remota. Fue fundada en 2008 y actualmente es propiedad de Microsoft. GitHub proporciona una interfaz gráfica amigable para trabajar con proyectos versionados con Git, además de herramientas colaborativas como seguimiento de issues (tareas o bugs), pull requests para revisión de cambios, integración continua (CI), y gestión de equipos.

En conjunto, Git y GitHub son una dupla central en el desarrollo moderno de software. Git ofrece el control de versiones y la solidez técnica, mientras que GitHub amplía esas capacidades al entorno colaborativo y social del desarrollo, donde múltiples personas pueden contribuir, revisar y mantener proyectos de forma organizada, desde pequeños scripts hasta sistemas complejos a escala global.
